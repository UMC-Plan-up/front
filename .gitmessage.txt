################ 타입 종류 ################
# feat : 새로운 기능 추가
# add : 파일 및 라이브러리 리소스 추가
# delete : 불필요한 파일 삭제
# fix : 오류 수정
# hotfix : 긴급 오류 수정
# style : 코드 수정(잔잔바리)
# refactor : 대규모 리팩토링(잔잔바리 수정은 style)
# chore : 버전 업데이트같이 코드 외 작업
# ui : UI 작업
# merge : 브랜치 병합
# move : 파일/디렉토리 이동 (내용 변경 X)
# rename : 파일/폴더 이름 수정
# docs : README 등 문서 작업
#
################ footer 종류 ################
# Close #이슈번호    : 이 커밋(PR)로 이슈가 완료되어 자동 종료
# Fix #이슈번호      : 버그 수정 커밋으로 이슈 해결 (자동 종료됨)
# Resolve #이슈번호  : 요청사항 완료 (자동 종료됨)
# Related to #이슈번호 : 해당 커밋이 해당 이슈 일부와 연관됨 (진행 중)
# Ref #이슈번호      : 참고만 했음, 코드 직접 연관 없음 (주로 문서, 논의, 아이디어 참고 등)
